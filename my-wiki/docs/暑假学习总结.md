# 暑假学习计划总结

暑假期间，我制定了系统的学习计划，包括理论学习、题目训练、赛事参与和经验分享四个方面，提升了自身的算法能力，同时还掌握了基础的 md 文档编写技能，并在 Notion 中完成了学习内容的整合梳理。以下是本次暑假学习计划的详细总结。

## 一、核心知识学习：算法基础

暑假的核心学习重点集中在数据结构与算法，针对基础的算法与数据结构进行了深入学习，掌握了基础原理，理解了其适用场景与优化思路，同时学习了 C++ lambda 表达式提升编程效率，具体内容如下：

| 知识模块          | 学习内容与核心收获                                           |
| ----------------- | ------------------------------------------------------------ |
| 差分              | 掌握了差分的核心思想 —— 通过 “前缀和的逆运算” 快速处理区间修改与查询问题，能熟练运用一维差分解决数组区间增减、二维差分处理矩阵区间更新等场景，大幅提升了这类问题的代码效率，避免了暴力解法的超时风险。 |
| 前缀和            | 深入理解前缀和与差分的互补关系，不仅能通过前缀和快速计算数组、矩阵的区间和，还能结合哈希表解决 “两数之和”“子数组和为 K” 等经典问题，同时掌握了前缀和在处理边界条件（如空区间、负数值）时的技巧。 |
| ST 表             | 学会了 ST 表（Sparse Table）的构建与查询逻辑，理解其基于 “倍增思想” 实现静态数组 RMQ（区间最值查询）的原理，能在 O (nlogn) 预处理后实现 O (1) 查询，明确了其 “静态数据适用、不支持动态修改” 的特性，可灵活用于无需更新的最值查询场景。 |
| 倍增算法          | 掌握倍增思想的核心 ——“以 2 的幂次为步长，减少查询次数”，除了在 ST 表中的应用，还学习了倍增在 LCA（最近公共祖先）、快速幂、二分查找优化等场景的用法，能通过预处理降低复杂问题的时间复杂度，例如将 LCA 查询从暴力的 O (n) 优化至 O (logn)。 |
| 线段树            | 系统学习了线段树的构建、单点更新、区间更新（懒标记）、区间查询（求和、最值）等操作，理解其 “分治” 与 “树形结构” 的本质，能独立实现线段树解决动态区间问题，例如 “区间增减 + 区间求和”“区间修改 + 区间最值查询”，同时掌握了线段树的空间优化（4 倍数组大小）与边界处理技巧。 |
| 树状数组          | 理解树状数组（Binary Indexed Tree）的底层二进制原理，掌握其单点更新、前缀和查询、区间和查询的实现方式，对比线段树明确了其 “代码简洁、空间占用小、适合前缀和 / 单点更新场景” 的优势，能熟练用树状数组解决 “逆序对统计”“动态频率查询” 等问题，同时学习了树状数组在高维场景（二维树状数组）的简单应用。 |
| C++ lambda 表达式 | 深入学习了 lambda 表达式的语法结构（捕获列表、参数列表、返回值类型、函数体），掌握了值捕获、引用捕获、全局捕获等不同捕获方式的区别与适用场景，能在算法题中灵活用 lambda 表达式简化代码，例如作为 sort 函数的自定义比较器、优先队列的排序规则，提升代码的可读性与编写效率。 |

## 四、经验分享：沉淀学习思路与 md 文档编写实践

为进一步梳理解题思路、加深对算法的理解，我不仅将典型题目的解题过程整理为题解，还在这一过程中学会了**基础的 md 文档编写**，通过 md 格式优化题解的排版与可读性，最终将题解发表在 CSDN、LeetCode、洛谷三大平台：

### （一）md 文档编写技能学习与应用

在题解创作中，我系统学习了 md 的核心语法，包括标题分级（# 至 ######）、列表（有序列表 1. 、无序列表 - ）、代码块（cpp ... ）、表格（| 表头 | 内容 |）、加粗（**重点内容**）、链接（[平台](链接地址)[名称](链接地址)）等，并将这些语法应用到题解编写中：

-   用**标题分级**划分题解结构（如 “一、题目分析”“二、思路推导”），让逻辑更清晰；

-   用**代码块**包裹 C++ 代码，搭配语法高亮（指定 cpp 语言），避免代码格式混乱，方便读者复制学习；

-   用**表格**对比不同算法的时间复杂度、适用场景（如差分 vs 线段树的优劣对比），让关键信息更直观；

-   用**加粗**标注核心考点、代码关键步骤（如 “注意：线段树懒标记需延迟更新”），帮助读者快速抓住重点。
